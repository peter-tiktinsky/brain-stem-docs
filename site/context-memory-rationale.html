The memory model

Why memory is split three ways and loaded as a single small index

By default a session forgets everything when it ends. The memory pillar makes durable knowledge survive: a small index that loads automatically at the start of every conversation, pointing to deeper notes read only when needed. Two platform facts shape almost everything here — the auto-loaded file is hard-capped (the first 200 lines or 25 KB, then silently dropped) and there is no include mechanism to pull in other files CC memory. An unbounded store doesn't fail loudly; it just goes invisible from the bottom up.

brain-stem choseoverbecauseground
Split durable memory into exactly three kinds — semantic, procedural, episodic One undifferentiated pile A single bucket gives a stale one-time correction the same retrieval weight as a durable fact. The split is the fifty-year memory-science consensus — episodic/semantic Tulving 1972, declarative/procedural Cohen & Squire 1980 — carried onto agents CoALA MemGPT. A fourth "reflective" type was rejected: reflection is a process, and its output is already a fact or a rule. convergence
One small auto-loaded index plus fetch-on-demand detail An unbounded or fully auto-loaded memory The auto-load file is hard-capped and content past the cap is silently dropped, so a fully auto-loaded store would be truncated invisibly CC memory. The only realizable shape is one hot MEMORY.md index plus cold, on-demand topic files. constraint
A small, curated memory A catch-everything store of every observation The more a catch-all grows, the lower its signal-to-noise falls. Mirrors MemGPT's small-core principle MemGPT; brain-stem asserts no measured decay rate, only the qualitative trend. convergence
Two scopes — per-project memory and a global rules/ folder A single "global project" memory directory Auto-memory is strictly per-git-root; the only surfaces that load in every session are ~/.claude/CLAUDE.md and ~/.claude/rules/ CC memory. A "global project dir" only loads from its own working directory, so it isn't actually global. constraint
Declare each note's kind at write time A separate program guessing the kind afterward The author already knows the fact's kind and can state it; guessing introduces silent misclassification you'd have to second-guess. The taxonomy is frozen at five filename prefixes times three retrieval types. convergence
Automated promotion writes only to rules/ Letting any writer touch your CLAUDE.md Your personal CLAUDE.md is authored once and yours alone to hand-edit — survivorship. Trust differs by direction: promoting project lessons up proposes-then-confirms; pulling from an external store is never automatic. design judgment
Episodic capture = a no-LLM chronicle plus one index pointer A per-session CoALA-STAR narrative CoALA A per-session STAR narrative would be static boilerplate with no retrieval value and would compete with the optional recall plugin. "Auto-generated survives, hand-maintained rots," so capture is harvested, not authored — a two-stage shape Zep LangMem independently endorse. convergence
Rotate the chronicle at a ~50 KB cap Calling 50 KB a "measured threshold" A no-LLM session-end hook can't tokenize, so bytes proxy a single-read token budget context engineering Lost in the Middle; the cap is a deliberate proxy, not a measured threshold. constraint
The load guard advises — it warns, never blocks Blocking the save on a cap breach Over-blocking a legitimate save is worse than a too-long index a human can trim; blocking trains people to disable the guard. A separate monitor only records status — it never auto-deletes curated memory OPA Gatekeeper ESLint. convergence
Count both raw and comment-stripped lines; gate on the larger A naive single-direction size check Otherwise bulky content could hide inside HTML comments to dodge the check — yet still cost load budget. Comment-stripping is not documented for MEMORY.md CC CLAUDE.md, so the safe default assumes comments cost budget. design judgment
Lock the section order now; document the overflow seam but build it later Building the cold-spill machinery now, or a calendar trigger The locked Semantic→Procedural→Episodic order makes the cold boundary knowable from the first index, so the future machinery drops in without re-deciding anything. The trigger is volume-gated, not calendar-gated — "wait 30 days" is arbitrary; the signal that matters is the index approaching the cap. constraint
Resolve the memory directory exactly as the harness does Reconstructing the slug with a naive path-replace Several near-misses would write where the running session never reads. The resolver must match the shipped binary — key by git root, realpath, honor autoMemoryDirectory first CC memory. constraint
Carry a machine-readable presence map alongside the human render Keeping only the inline render, or only the map Inline-only forces machines to parse rendered prose Tolerant Reader; map-only loses the human "checked-and-absent vs broken-link" disambiguation. Serve both audiences without coupling them. convergence
For the live corpus, split memory per-project Retaining the shared corpus, a hybrid, or tiering by access frequency The 25 KB cap makes one shared corpus permanently lossy — only ~37% of entries load; per-project split is the only candidate that structurally kills the dark fold. Precondition: the always-on rules/ tier must stay small and universal #21858/#25562. constraint

Context & Library

Why context is structured as Workshop, Library, and promotion

Context is the durable material that frames the work and records its state — distinct from the always-on operational memory that governs how the assistant acts. brain-stem splits it across an ephemeral Workshop to think in, a universal Library of reusable reference, and a promotion path between them. The load-bearing thread: auto-generated surfaces survive while hand-maintained ones rot. The rows below are the ones that belong to context & memory; the binder, _situating.md, and launch-directory identity are addressed in the project management rationale, and the work/ deliverable layer in the work-folder guidebook.

brain-stem choseoverbecauseground
Split context from always-on operational memory One undifferentiated "knowledge" pile You can't reason about a store that conflates how-to-act with what-happened with what-frames-the-topic — it fills up, drifts, and is abandoned. This is the documented capture-then-drift / maintenance-collapse pattern — the Collector's Fallacy second brain. convergence
Home the durable surfaces in the plan area Homing them inside the tool's own config folder (~/.claude/) That folder is protected by a rule that blocks edits to sensitive files, so a script writing reference there fails unreliably — and "a store you cannot reliably write to is not a store." constraint
Home surfaces outside the synced vault; expose a read-only symlink view Homing the real surfaces inside the notes vault A notes folder is one setting away from cloud sync, where silent edit-failures are a known hazard #52493, and Obsidian symlink hazards (git syncs the target, the watcher misses external edits) compound it Obsidian symlinks. constraint
The Workshop is ephemeral — never loaded, never indexed One folder holding raw notes and finished reference Forcing structure onto scratch paper only slows the work: "the Workshop is where you think; the Library is what you keep." Surveys of real setups found no durable two-in-one store; and it matches what Zettelkasten practitioners do — discard a project's scratch notes once the work closes Zettelkasten. convergence
Promotion is a scrub-transformation, gated by a proposed diff A verbatim copy from project to Library A "universal" article still carrying one project's names is "project leftovers on the wrong shelf"; a draft that is nothing but project specifics has nothing left after scrubbing, so it never promotes. The scrub ships advisory-first, with a human diff backstop. design judgment
The Library reuse hint is an advisory prompt-time nudge Full-loading the shelf; a standing "check first" rule; an after-action check; a block Full-load defeats loading discipline; a standing rule rots #33603 #56393; an after-action check fires too late and can't inject context #18427. A prompt-time hook is the only vehicle that fires before research begins without blocking — and a hook beats skill auto-detection, which went un-invoked 56% of the time Vercel eval. constraint convergence
